# 阶段一复习与练习方案

> 用法：放进你的 `llm-learning` 仓库根目录，当 checklist 用。
> 核心理念：**不专门停下来复习，把巩固拆进每天的日常**。工具类知识靠"反复用"巩固，概念类知识靠"反复答为什么"巩固，防遗忘靠"间隔重复"。

---

## 三个固定动作（养成习惯就够了）

### ① 每天开工前 5 分钟：空手默答"为什么"
不看任何资料，口头或在 LOG.md 里回答当前进度对应的"为什么清单"（见下）。答得流畅 → 直接开始新内容；卡壳 → 翻回去看一眼再开始。**这是巩固概念最高效的方式：主动回忆比被动重读强得多。**

### ② 每天收工：记 LOG + 每 3 天一次"不看抄"挑战
- 平时：正常写 LOG.md（学了什么 / 卡在哪 / 明天补什么）
- 每隔约 3 天：挑一个已学功能，**关掉旧代码**，只看官方文档，从零把它搭出来。搭不出来的地方就是你的真实薄弱点，针对性补。

### ③ 每周日 30 分钟：过一遍本周全部"为什么"清单
一周前学的东西此时正好临近遗忘，过一遍即可长期记住。这一步用掉的 30 分钟，比考前临时抱佛脚省事得多。

---

## 概念自测：Day 1 的"为什么清单"

能用自己的话答出来就算掌握（不要求逐字，要求讲清逻辑）：

- [ ] 为什么把 base_url 换成 DeepSeek 的地址，同一套代码就能调通？
- [ ] system / user / assistant 三种角色各自的作用是什么？
- [ ] 模型本身没有记忆，那"多轮对话能记住上文"是怎么实现的？
- [ ] 为什么 API key 不能写进代码、不能传到 GitHub？正确的做法是哪几步？
- [ ] .env、.venv、pyproject.toml、.gitignore 各是干嘛的？
- [ ] temperature 调高和调低，输出分别会怎样？各适合什么场景？
- [ ] max_tokens 是"让模型精简回答"吗？它到底做了什么？
- [ ] git add / commit / push 三步分别在干嘛？

> 进度往后走时，每天把当天的"为什么"补进这份清单，滚动累积。

## 动手练习：三档难度，按状态选

### 入门档（巩固肌肉记忆，5–10 分钟）
- [ ] 不看旧代码，凭记忆写出读取 .env 里 key 的三行（load_dotenv / os.getenv）
- [ ] 不看，敲出 git add → commit → push 三连

### 标准档（"不看抄"挑战，20–30 分钟）
- [x] 只看 DeepSeek 官方文档，从零写一个**多轮对话脚本**：用 while 循环接收输入，每轮把模型回复以 assistant 角色加回 messages，验证它能记住上文（比如先问"我叫小明"，再问"我叫什么"）
- [x] 这个脚本正是 Day 1 验收标准，做出来 = Day 1 真过关 ✅

### 挑战档（融会贯通，30–45 分钟，有余力再做）
- [x] 给多轮对话脚本加一个"输入 clear 就清空历史重新开始"的功能
- [x] 加一个统计：每轮打印这次用了多少 token（提示：response.usage 里有）
- [x] 把 temperature 做成启动时可选（让用户选"严谨"还是"创意"模式）

## 间隔重复时间表（防遗忘，照着勾就行）

| 时间点 | 复习什么 | 花多久 |
|--------|----------|--------|
| 学完当天 | 写 LOG + 答当天"为什么" | 10 分钟 |
| 第 2 天 | 开工前默答前一天"为什么" | 5 分钟 |
| 第 4 天 | "不看抄"挑战一个功能 | 20 分钟 |
| 每周日 | 过一遍本周全部"为什么" | 30 分钟 |
| 阶段一结束 | 集中过所有清单 + 准备面试题 | 半天 |

---

## 判断"巩固到位"的标准

不是能默写代码（那不必要），而是同时满足：
1. **讲得清**：合上电脑能向人解释那些"为什么"
2. **搭得出**：看着文档能独立把功能实现，每行都知道为什么这么写
3. **不易忘**：隔几天再问，原理还在脑子里

三条都满足，这部分就真正是你的了，可以放心往后推进。
